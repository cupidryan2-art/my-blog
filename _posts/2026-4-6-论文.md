### Transformer架构

## Input Representation

1.1 

核心定义在 Transformer 中，词嵌入是将离散的 Token（单词或子词）映射为连续向量的空间。

论文中使用的嵌入维度 $d_{model} = 512$。

1.2 原理详解

线性映射：模型通过一个可学习的权重矩阵 $W_e \in \mathbb{R}^{V \times d_{model}}$（其中 $V$ 是词表大小）将 Token ID 转换为 512 维的浮点向量。

权重共享 (Weight Sharing)：论文中，两个 Embedding 层（Encoder 和 Decoder 的输入）以及最后的 Pre-softmax 线性层共享相同的权重矩阵。

缩放处理 (Scaling)：在将向量传给下一层前，Embedding 向量会乘以 $\sqrt{d_{model}}$。

为什么要乘以 $\sqrt{d_{model}}$？

这是为了平衡 Positional Encoding（位置编码）的影响，确保 Embedding 的原始语义信号在相加后不会被位置信号完全掩盖。

2. Positional Encoding (位置编码)

2.1 核心动机 (The "Why")Transformer 

抛弃了循环神经网络（RNN）的顺序结构，采用全并行计算。为了让模型感知单词在序列中的绝对位置和相对位置，必须手动注入位置信息。

2.2 数学实现

论文采用了基于正弦和余弦函数的固定编码方式：$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$$$PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$$pos$：Token 在句子中的物理位置（如 0, 1, 2...）。$i$：向量维度的索引（从 0 到 $d_{model}/2$）。$d_{model}$：模型维度（512）。

2.3 关键特性

相对位置表达：由于三角函数的特性，对于任何固定偏移量 $k$，$PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的线性函数。这使得模型能够轻易学习到词语间的相对距离。

长度外推性：这种周期性函数允许模型在推理时处理比训练时更长的序列。

3. 最终输入构造 (Data Flow)最终进入 Encoder/Decoder 模块的向量 $X_{final}$ 是两者的直接相加：$$X_{final} = \text{Embedding}(Token) \cdot \sqrt{d_{model}} + \text{Positional Encoding}$$
